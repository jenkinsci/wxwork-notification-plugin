# 前置配置

以下配置同时适用于 FreeStyle 和 Pipeline，正式使用前建议先完成。

## 1. 配置企业微信群机器人

进入 `管理 Jenkins -> 系统 -> 企业微信群机器人 -> 新增`，填写机器人信息：

![全局设置](../images/stage1-1.png)

![系统设置](../images/stage1-2.png)

![配置机器人](../images/stage1-3.png)

- **ID**：机器人唯一标识，FreeStyle 和 Pipeline 均通过该值引用
- **名称**：机器人显示名称，主要用于 Jenkins 页面展示
- **Webhook URL**：企业微信群机器人提供的 Webhook 地址

配置完成后，建议先点击测试按钮验证连通性，再保存配置。

![测试机器人](../images/stage1-4.png)

## 2. 配置用户手机号（可选）

如果需要使用 `@构建执行人` 功能，需要在 Jenkins 用户资料中填写企业微信手机号：

`点击用户名 -> 配置 -> 企业微信手机号`

![配置手机号](../images/stage2-1.png)

注意事项：

- 手机号必须与企业微信成员绑定手机号一致
- 否则 `@构建执行人` 不会生效
- 该配置只影响按用户自动提醒，不影响手动填写手机号列表的 `@` 功能

## 配置建议

- 机器人 `ID` 建议使用稳定、易识别的英文标识，例如 `ci-main`、`release-bot`
- 如果需要面向不同群发送通知，建议预先配置多个机器人，按任务分别选择
- 初次接入时先用测试消息验证权限、Webhook 地址和群可用性

## 常见使用关系

- FreeStyle 页面下拉选择的是机器人 `名称`
- Pipeline `wxwork(robot: '...')` 传入的是机器人 `ID`
- `@构建执行人` 依赖 Jenkins 用户资料中的企业微信手机号
- `@ 用户（手机号）` / `at` 可直接填写手机号列表，不依赖用户资料
